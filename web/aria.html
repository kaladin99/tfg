Què és WAI-ARIA i quan convé usar-la

WAI-ARIA (Accessible Rich Internet Applications) és un conjunt de roles, estats i propietats que complementen l’HTML per comunicar millor la interfície a les tecnologies d’assistència (p. ex. lectors de pantalla). La norma d’or és: primer, HTML natiu; només si no és suficient, afegeix ARIA per descriure rols, estats i relacions.

aria-label — Nom accessible explícit

Proporciona un accessible name quan el control no té text visible suficient (p. ex. botó d’icona).

Codi de l’exemple (aria-label) 
<button type="button" aria-label="Obrir menú principal">
  <span aria-hidden="true">☰</span>
</button>

aria-labelledby — Nom referenciat a text visible

Menú principal
Codi de l’exemple (aria-labelledby) 
<span id="txt-menu-principal">Menú principal</span>
<button type="button" aria-labelledby="txt-menu-principal btn-menu-ref" id="btn-menu-ref">
  <span aria-hidden="true">☰</span>
</button>

aria-describedby — Descripció/ajuda complementària

No compartirem mai el teu correu.
Codi de l’exemple (aria-describedby) 
<label for="email1">Correu electrònic</label>
<small id="ajuda-email1">No compartirem mai el teu correu.</small>
<input id="email1" type="email" aria-describedby="ajuda-email1">

aria-controls i aria-expanded — Mostrar/ocultar contingut

Resposta de l’exemple. Aquest bloc es mostra i s’oculta amb el botó.

Codi de l’exemple (aria-controls + aria-expanded) 
<button type="button" aria-controls="faq-aria" aria-expanded="false" id="btn-faq-aria">Mostra resposta</button>
<div id="faq-aria" hidden>
  <p>Resposta de l’exemple. Aquest bloc es mostra i s’oculta amb el botó.</p>
</div>

<script>
(function () {
  const btn = document.getElementById('btn-faq-aria');
  const panel = document.getElementById('faq-aria');
  if (btn && panel) {
    btn.addEventListener('click', () => {
      const expanded = btn.getAttribute('aria-expanded') === 'true';
      btn.setAttribute('aria-expanded', String(!expanded));
      panel.hidden = expanded;
      btn.textContent = expanded ? 'Mostra resposta' : 'Oculta resposta';
    });
  }
})();
</script>

aria-live — Anuncis de canvis dinàmics

Codi de l’exemple (aria-live) 
<div aria-live="polite" id="live-polite-aria"></div>
<button type="button" id="btn-afegir-aria">Afegir element</button>

<script>
(function () {
  const live = document.getElementById('live-polite-aria');
  const btn = document.getElementById('btn-afegir-aria');
  let n = 0;
  if (btn && live) {
    btn.addEventListener('click', () => {
      n += 1;
      live.textContent = `S'ha afegit l'element ${n}.`;
    });
  }
})();
</script>

Altres atributs útils i bones pràctiques

  • aria-current: marca l’element actual d’una navegació (p. ex. pàgina activa).
  • aria-hidden: oculta l’element a tecnologia d’assistència (no visualment). No l’apliques a contingut necessari.
  • aria-pressed: estat d’un botó commutable (toggle).
  • aria-disabled: indica deshabilitació quan no pots usar disabled natiu.
  • role: sols quan l’element natiu no cobreix el cas (p. ex. role="dialog" en un contenidor personalitzat).
Exemple ràpid amb aria-current 
<nav aria-label="Paginació">
  <a href="?p=1">1</a>
  <a href="?p=2" aria-current="page">2</a>
  <a href="?p=3">3</a>
</nav>
